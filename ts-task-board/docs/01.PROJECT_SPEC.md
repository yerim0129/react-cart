# ts-task-board — 프로젝트 스펙 & 학습 가이드

> TypeScript + React를 마스터하기 위한 연습 전용 프로젝트  
> 모노레포 구조의 `ts-task-board/` 디렉토리에 위치

---

## 모노레포 구조

```
monorepo/
├── react-cart/          # 기존 쇼핑몰 프로젝트 (수정 금지)
└── ts-task-board/       # TypeScript + React 연습 프로젝트 ← 여기
```

---

## 데이터 구조

### User

| 필드  | 타입                | 설명        |
| ----- | ------------------- | ----------- |
| id    | string              | 고유 식별자 |
| name  | string              | 사용자 이름 |
| email | string              | 이메일      |
| role  | `"ADMIN" \| "USER"` | 권한        |

### Task

| 필드        | 타입                                | 설명        |
| ----------- | ----------------------------------- | ----------- |
| id          | string                              | 고유 식별자 |
| title       | string                              | 제목        |
| description | string                              | 설명        |
| status      | `"TODO" \| "IN_PROGRESS" \| "DONE"` | 진행 상태   |
| priority    | `"LOW" \| "MEDIUM" \| "HIGH"`       | 우선순위    |
| assignee    | User                                | 담당자      |
| dueDate     | Date                                | 마감일      |
| createdAt   | Date                                | 생성일      |
| updatedAt   | Date                                | 수정일      |

### Category

| 필드  | 타입   | 설명        |
| ----- | ------ | ----------- |
| id    | string | 고유 식별자 |
| name  | string | 카테고리명  |
| color | string | 색상 코드   |

## 학습 로드맵

### ✅ Step 1 — Interface 기초

- `src/interfaces/` 폴더 생성
- `IUser.ts`, `ITask.ts`, `ICategory.ts` 작성
- 학습 포인트: 필수 속성, optional(`?`), union type (`"TODO" | "IN_PROGRESS"`)

### ✅ Step 2 — Interface 상속

- `IBaseEntity` 공통 베이스 인터페이스 생성
- `ITask`, `IUser` 가 `IBaseEntity` 를 `extends` 하도록 리팩터링
- `ISearchTask`, `ISearchUser` 검색 조건 인터페이스 작성

### ✅ Step 3 — 제네릭 Interface

- `PagedResponse<T>` 제네릭 인터페이스 작성
- `PagedResponse<ITask>`, `PagedResponse<IUser>` 로 활용 실습

### ✅ Step 4 — MSW로 API 모킹

- MSW 세팅: `src/mocks/handlers.ts`, `src/mocks/browser.ts` 작성
- Task / User CRUD 핸들러 작성 (GET / POST / PATCH / DELETE)
- 학습 포인트: Service Worker 기반 모킹 원리, `http.get`, `HttpResponse` 사용법

### ✅ Step 5 — API 레이어 연결

- `src/apis/task.ts` — `fetchTasks(searchParam: ISearchTask)` 함수 작성
- 컴포넌트에서 TanStack Query로 데이터 패칭 후 화면 표시
- 학습 포인트: `useQuery`, `useMutation`, `queryKey` 설계

### ✅ Step 6 — 컴포넌트에서 Interface 활용

- props 타입 지정 실습: `function TaskCard({ task }: { task: ITask })`
- VSCode 자동완성 및 타입 에러 체험

## 진행 원칙

1. **단계별 진행** — 한 번에 전체를 만들지 않고, 이해 확인 후 다음 단계 진행
2. **설계 이유 먼저** — 코드 작성 전 "왜 이렇게 설계하는지" 설명
3. **직접 작성 유도** — 힌트 제공 후 직접 작성 기회 부여
4. **에러 분석** — 발생한 에러를 TypeScript 관점에서 원인과 함께 설명

## 초기 세팅 요청

```bash
# 1. ts-task-board 폴더에 Vite + React + TypeScript 프로젝트 초기화
npm create vite@latest ts-task-board -- --template react-ts

# 2. 의존성 설치
cd ts-task-board
npm install
npm install @tanstack/react-query
npm install -D tailwindcss postcss autoprefixer
npm install msw --save-dev

# 3. src/interfaces/ 폴더 생성 후 Step 1 시작
```
